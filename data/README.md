# Data

Everything in `data/sample/` is synthetic, generated by
`scripts/make_samples.py` with fixed seeds. Each `.npz` holds one field:
`noisy` (count image), `clean` (expected counts), `positions` (ground-truth
column centres), `weights` (relative column weights) and `dose`. No
external dataset is required anywhere in this repository; the simulator
is the dataset.

Regenerate the samples at any time:

```
python scripts/make_samples.py
```

## Bringing your own STEM data

The tools accept real images through `stemdenoise.io.load_image`
(`.npy`, `.npz`, `.png`, `.tif`, `.tiff`; multi-channel frames are
averaged to one channel):

```
stemdenoise denoise my_frame.tif --method cnn --checkpoint models/unet_supervised.pt
stemdenoise denoise my_frame.tif --method nlm
```

Two caveats to read before trusting the output:

1. The Poisson noise model and the CNN normalization assume roughly
   count-scaled data. For raw counts this holds; for rescaled 8/16-bit
   exports, `stemdenoise.io.estimate_dose` maps the image onto the
   training normalization approximately, and results should be read
   qualitatively.
2. The CNN was trained only on the two synthetic lattices described in
   the model card. On very different structures (amorphous material,
   large defects, strong scan distortion) prefer the variance-stabilized
   classical methods, which assume nothing about structure.

No real micrograph is committed to this repository, because none with an
unambiguous license was part of this build. If you want a public test
image, electron-microscopy images on Wikimedia Commons with explicit CC
licenses are a reasonable source; keep any such file and its attribution
out of the MIT-licensed tree (for example in `data/real/` with its own
README) if you add one locally.
